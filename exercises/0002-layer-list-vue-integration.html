图层列表(Vue + TS)

目标:把第八课(tsconfig、defineProps/defineEmits、共享类型放哪)在父子组件之间多写几遍,巩固模式,不引入新概念。

背景

在你已有的 learn_ts_project 里继续写。做一个简化版"图层列表":App.vue 管理一份图层数据数组,LayerCard.vue(你已经写好一版了)展示单个图层并在用户点击时触发"切换可见性"事件,App.vue 接收事件后更新对应图层在数组里的状态。

要求

1. 定义一个共享类型 LayerItem

包含 name: string、visible: boolean、opacity?: number。想一想:这个类型会被 App.vue 和 LayerCard.vue 同时用到,按第八课讲的判断标准,应该放在组件里还是拆到 src/types/ 下?拆的话文件名和 import 路径自己定。

2. 改造 LayerCard.vue,让它显示真实内容

你之前写的 LayerCard.vue 按钮是空的。现在用 props 把 name、visible、opacity 显示在 <template> 里(比如 <p>{{ name }} - {{ visible ? '可见' : '隐藏' }}</p>),按钮文字也可以按 visible 动态变化(比如显示"隐藏"或"显示")。Props 直接用 LayerItem(或者继续单独定义,想一下两者取舍)。

3. App.vue 维护一个图层数组

用 ref<LayerItem[]>([...]) 初始化 2-3 条图层数据,用 v-for 渲染多个 LayerCard,记得给 v-for 加 :key。每个 LayerCard 的 props 要从数组里对应的那一项传进去(注意哪些要加 :,哪些不用)。

4. 接收事件,更新数组里对应的那一项

LayerCard 触发 toggle-visible 时,App.vue 要找到数组里"是哪一个图层"(比如按 name 匹配,或者用下标),把它的 visible 改成事件带过来的新值,而不是把整个数组重新赋值成一份新数据。

验收标准

跑起来之后,页面上能看到 2-3 个图层卡片,点击任意一个的按钮,只有那一个卡片的可见状态文字变化,其他不受影响,刷新页面后恢复初始数据(不要求做持久化)。

卡住了?点开看提示(不是答案)

1. v-for 里传 props 给子组件,和之前直接手写一个 <LayerCard name="roads" :visible="true" /> 本质一样,只是值从数组元素里取,比如 :name="layer.name"、:visible="layer.visible"。
2. 找到数组里"是哪一个"最简单的办法是 layers.value.findIndex(l => l.name === xxx),或者在 v-for 里直接用下标 (layer, index) in layers,事件处理时把 index 一起传出去。
3. emit 触发时能不能一次带两个参数?回想第八课 defineEmits 元组语法,一个事件可以对应多个位置的参数,比如 'toggle-visible': [index: number, value: boolean]。
4. 更新数组某一项的 visible,不要整个替换数组,只改那一项:layers.value[index].visible = newValue 就够了,Vue 的响应式能追踪到。
