综合练习 1

图层管理器

目标:把前七课(interface、联合类型收窄、class/抽象类、泛型、工具类型、类型断言、enum)拼进同一个小系统里,检验能不能组合使用,不是学新概念。

背景

做一个简化版的地图图层管理器:能装不同类型的图层(矢量/栅格),能批量渲染,能局部更新配置,能从"外部数据源"(模拟成 unknown)加载图层配置。

要求

1. 用 enum 定义图层类型

定义一个字符串枚举 LayerKind,包含 Vector、Raster 两种取值。

2. 用抽象类定义图层基类(第四课)

MapLayer 是抽象类,至少包含:name: string、kind: LayerKind、抽象方法 render(): void。用参数属性简写声明字段。

3. 写两个具体子类

VectorLayer 和 RasterLayer 继承 MapLayer,各自实现 render(),并各自带一个私有字段(比如 VectorLayer 的 sourceUrl,RasterLayer 的 opacity)。构造函数里要正确处理 super() 调用顺序。

4. 用泛型 class 写一个分组容器(第五课)

LayerGroup<T extends MapLayer>,包含 add(layer: T): void 和 renderAll(): void(遍历调用每个图层的 render()——这里会用到抽象类的多态)。

5. 用联合类型 + 收窄写一个"打印图层详情"的函数(第三课)

写一个函数,参数类型是 VectorLayer | RasterLayer,内部用 kind 字段(或者 instanceof)判断具体类型,分别打印不同的详情(矢量图层打印 sourceUrl,栅格图层打印 opacity)。

6. 用工具类型写一个"更新配置"的函数(第六课)

定义一个 LayerConfig interface(包含 name、visible 两个字段),写一个函数 updateConfig(current: LayerConfig, changes: Partial<LayerConfig>): LayerConfig,返回合并后的新配置(提示:可以用对象展开 { ...current, ...changes })。

7. 用类型断言写一个"从外部数据源加载配置"的函数(第七课)

写一个函数 loadConfigFromSource(): unknown,内部随便 return 一个符合 LayerConfig 形状的对象。调用处用 as LayerConfig 断言,再传给 updateConfig 用。

验收标准

最终应该能跑通类似这样的调用(不要求一字不差,能表达出同样的组合关系就算达标):

const vectorGroup = new LayerGroup<VectorLayer>();
vectorGroup.add(new VectorLayer("roads", LayerKind.Vector, "/data/roads.geojson"));
vectorGroup.renderAll();

const rasterGroup = new LayerGroup<RasterLayer>();
rasterGroup.add(new RasterLayer("dem", LayerKind.Raster, 0.8));
rasterGroup.renderAll();

describeLayer(new VectorLayer("roads", LayerKind.Vector, "/data/roads.geojson"));
describeLayer(new RasterLayer("dem", LayerKind.Raster, 0.8));

const raw = loadConfigFromSource();
const config = raw as LayerConfig;
const updated = updateConfig(config, { visible: false });
console.log(updated);
卡住了?点开看提示(不是答案)

1. 抽象类的字段和泛型约束的字段要对得上——LayerGroup<T extends MapLayer> 里的 T 就是 MapLayer 或它的子类。
2. 子类 constructor 里参数顺序随意,但记得 super(name, kind) 之类的调用必须在使用 this 之前。
3. 第 5 步判断具体类型,可以用 instanceof VectorLayer,也可以在 MapLayer 上加一个 kind 字段用 if (layer.kind === LayerKind.Vector) 判断——两种都行,想想哪种更符合"可辨识联合"的思路。
4. 第 7 步的 unknown 是故意的,模拟"从外部拿到的数据,TS 不知道具体类型",这样才有理由用 as。
