Class 语法与访问修饰符

配合《Class 语法基础与 TS 的类型能力》食用

JS class 基础

class MapLayer {
  constructor(name, visible) {
    this.name = name;
    this.visible = visible;
  }
  toggle() {
    this.visible = !this.visible;
  }
}

class VectorLayer extends MapLayer {
  constructor(name, visible, sourceUrl) {
    super(name, visible); // 必须先调用 super()
    this.sourceUrl = sourceUrl;
  }
}

访问修饰符(编译期,运行时会被擦除)

class MapLayer {
  public name: string;        // 默认,任何地方都能访问
  private visible: boolean;   // 只能在本类内部访问
  protected sourceUrl: string; // 本类 + 子类可访问
}

参数属性(简写)

class MapLayer {
  constructor(
    public name: string,
    private visible: boolean
  ) {}
  // 等价于手写字段声明 + constructor 里的 this.xxx = xxx
}

抽象类(abstract)

abstract class MapLayer {
  abstract render(): void;   // 只有签名,子类必须实现
  describe(): void { /* 普通方法,可以有实现,直接继承 */ }
}

class VectorLayer extends MapLayer {
  render(): void { /* 具体实现 */ }
}

new MapLayer();       // 报错:不能实例化抽象类
new VectorLayer();    // 合法

implements(实现 interface)

interface Renderable {
  render(): void;
}

class RasterLayer implements Renderable {
  render(): void { /* 必须实现,否则报错 */ }
}
记住
private/protected/readonly 都只是编译期的约定,编译成 JS 后会被擦除,运行时和普通属性没有区别。真正运行时私有要用 JS 原生的 #field 语法。
