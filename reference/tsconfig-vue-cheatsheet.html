tsconfig 与 Vue + TypeScript 集成

配合《tsconfig 与 Vue + TypeScript 集成》食用

tsconfig 关键字段

{
  "compilerOptions": {
    "target": "ES2020",       // 编译产物的 JS 版本
    "module": "ESNext",       // 编译产物的模块写法
    "strict": true,           // 严格检查总开关,新项目建议从第一天开
    "skipLibCheck": true      // 跳过第三方库 .d.ts 自身的类型检查
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules"]
}

Vue SFC 里的 props / emits

<script setup lang="ts">
interface Props {
  title: string
  count?: number        // ? 表示可选
}
const props = defineProps<Props>()

const emit = defineEmits<{
  update: [value: number]  // 事件参数用数组(元组)表达
  close: []                 // 无参数事件
}>()
</script>

共享类型放哪

// src/types/layer.ts
export interface LayerConfig {
  name: string
  visible: boolean
}
// 组件里: import type { LayerConfig } from '@/types/layer'

局部类型就近写在组件里;被 3+ 处引用才拆到 src/types/。

类型检查命令

vue-tsc --noEmit   # 不是 tsc,因为 tsc 不认识 .vue 文件结构
记住
strict 越早开越好,老项目后补代价很高。defineProps<T>() 泛型写法能表达可选字段/联合类型等运行时写法很难表达的结构。
