interface 与 type

配合《interface / type —— 给对象结构起名字》食用

基本写法

// interface
interface User {
  name: string;
  age: number;
}

// type
type User = {
  name: string;
  age: number;
};

// 两种写法用起来完全一样:
function printUser(user: User): void {
  console.log(user.name, user.age);
}

可选属性 ?

interface User {
  name: string;
  age?: number; // 可以不传
}

printUser({ name: "Ella" }); // 合法

只读属性 readonly

interface Point {
  readonly lng: number;
  readonly lat: number;
}

const p: Point = { lng: 116.4, lat: 39.9 };
p.lng = 100; // 编译报错:Cannot assign to 'lng' because it is a read-only property.

联合类型 |(先混个脸熟,下节课细讲)

interface Feature {
  geometryType: "Point" | "LineString" | "Polygon";
}

interface 继承 / type 交叉

// interface 用 extends
interface BaseLayer {
  id: string;
  visible: boolean;
}
interface VectorLayer extends BaseLayer {
  features: number;
}

// type 用 &(交叉类型)
type BaseLayer = { id: string; visible: boolean };
type VectorLayer = BaseLayer & { features: number };

interface 还是 type?

描述对象/类的结构两者都行,团队统一用一种即可;官方推荐优先 interface
需要联合类型、元组、函数类型别名只能用 type
同名声明自动合并(declaration merging)只有 interface 支持
记住
起名字之后,类型可以像变量一样被复用、被继承、被传给函数参数——这是它们比"内联对象类型"更强大的地方。
