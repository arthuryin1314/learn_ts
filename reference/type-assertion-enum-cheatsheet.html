类型断言(as)与枚举(enum)

配合《类型断言(as)与枚举(enum)》食用

类型断言 as

const raw: unknown = JSON.parse(text);
const config = raw as { name: string; visible: boolean };
// 只在编译期生效,不做运行时检查——断言错了运行时可能直接崩

as const

const kind = "vector" as const; // 类型是字面量 "vector",不是 string

enum

enum LayerKind {
  Vector = "vector",
  Raster = "raster",
}

const kind: LayerKind = LayerKind.Vector;
Object.values(LayerKind); // 运行时真实存在的对象,可以遍历

enum vs 字符串联合类型

type LayerKindUnion = "vector" | "raster"; // 纯编译期,零运行时开销,擦除
enum LayerKind { Vector = "vector", Raster = "raster" } // 运行时真实对象
记住
类型收窄(if/typeof)是"运行时检查换信任",as 是"没检查、直接要求信任"——能收窄就不要用 as。enum 是本课目前唯一一个编译后不会被完全擦除的类型概念。
