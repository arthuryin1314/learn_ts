# 掌握了编译时类型检查 vs 运行时行为的区别

用户最初误以为"类型错误会在运行时抛出异常"，经过对比 `add(1, "2")` 报错的例子和自己动手实践后，能准确表述出正确理解：类型检查只发生在编译/写代码阶段，一旦编译通过，运行时和普通 JS 完全一样，不会因类型做任何检查。

## Evidence
- 自己编写 `printUser` 函数时犯了两个典型新手错误：把字符串字面量类型 `'string'` 误用成类型名 `string`；对象字面量语法写成 `{'Ella','35'}` 缺少键名。经提示后独立改正。
- 改正后自己验证：把 `age` 传字符串会在编译期报错 `Type 'string' is not assignable to type 'number'`，并能正确复述"原来规定为数字类型的变量给了字符串"——说明理解已经内化，不是背答案。

## Implications
- 后面讲 interface/type、泛型时，可以直接依赖"编译时 vs 运行时"这个心智模型，不需要重新建立。
- `string`（类型名）vs `'string'`（字符串字面量类型）的混淆是需要留意的坑，后续如果涉及字面量类型（literal types）时可以主动提一下，帮助避免同类错误再犯。
