第 6 课

工具类型(Utility Types)

目标:掌握 Partial、Required、Pick、Omit、Readonly、Record——这些是"从已有类型派生新类型"的标准工具,不用每次都手写一个新 interface。

问题:已经有一个类型了,但需要它的"变体"

第四课定义过 VectorLayer 需要 name、visible、sourceUrl 三个字段。假设现在要写一个"更新图层配置"的函数,调用者可能只想改其中一两个字段,其余保持不变。如果手写一个新 interface,所有字段都要重复声明一遍,而且原类型一旦改动,这个新类型很容易忘记同步。工具类型就是用来"从已有类型派生"的,不用手写重复的字段列表。

Partial<T>:把所有字段变成可选

interface LayerConfig {
  name: string;
  visible: boolean;
  sourceUrl: string;
}

function updateLayer(id: string, changes: Partial<LayerConfig>): void {
  // changes 里 name/visible/sourceUrl 全部变成可选,调用者传哪个改哪个
}

updateLayer("roads", { visible: false });              // 合法,只传一个字段
updateLayer("roads", { name: "roads-v2", visible: true }); // 合法,传多个
updateLayer("roads", {});                                // 合法,一个都不传

Partial<LayerConfig> 相当于把 LayerConfig 里的每个字段都自动加上 ?,等价于手写: 

interface LayerConfigPartial {
  name?: string;
  visible?: boolean;
  sourceUrl?: string;
}

Required<T>:反过来,把所有字段变成必填

如果原类型里某些字段本来是可选的(比如 sourceUrl?: string),但某个场景下必须全部提供,用 Required: 

interface LayerConfigLoose {
  name: string;
  sourceUrl?: string; // 本来可选
}

function saveLayer(config: Required<LayerConfigLoose>): void {
  // 这里 sourceUrl 变成必填,不传会报错
}

saveLayer({ name: "roads" }); // 报错:缺少 sourceUrl

Pick<T, K>:只挑几个字段出来

type LayerSummary = Pick<LayerConfig, "name" | "visible">;
// 等价于 interface LayerSummary { name: string; visible: boolean; }

const summary: LayerSummary = { name: "roads", visible: true };
// summary.sourceUrl; // 报错,Pick 之后这个字段根本不存在

第二个类型参数 K 是字段名组成的联合类型(字符串字面量),必须是 T 里真实存在的字段名,写错字段名 TS 会直接报错。

Omit<T, K>:和 Pick 相反,排除几个字段

type LayerWithoutSource = Omit<LayerConfig, "sourceUrl">;
// 等价于 interface LayerWithoutSource { name: string; visible: boolean; }

Pick 和 Omit 做的是同一件事的两种表达方式:字段少的时候用 Pick 直接列出要留下的,字段多、只想去掉一两个的时候用 Omit 更省事。

Readonly<T>:把所有字段变成只读

type FrozenLayerConfig = Readonly<LayerConfig>;

const config: FrozenLayerConfig = { name: "roads", visible: true, sourceUrl: "/data/roads.geojson" };
config.name = "roads-v2"; // 报错:Cannot assign to 'name' because it is a read-only property.

这里的 readonly 和第一课/第四课提到的 readonly 关键字是同一个语义,只是 Readonly<T> 一次性把所有字段都标成 readonly,不用逐个字段手写。

Record<K, V>:批量声明"键是 K 类型、值是 V 类型"的对象

第五课在 Feature<G> 里已经预告过,这里正式讲:

const layerVisibility: Record<string, boolean> = {
  roads: true,
  lake: false,
  buildings: true,
};

layerVisibility.roads;   // boolean
layerVisibility.anything; // 也是 boolean 类型(TS 不知道具体有没有这个 key,但类型上认为是 boolean)

Record<K, V> 常用在"key 的具体名字不重要或者不固定,但每个 value 的类型都一样"的场景,比如 GeoJSON properties 字段(key 是任意字符串,value 类型不确定就用 unknown):Record<string, unknown>。

组合使用

工具类型可以叠加,比如"只挑两个字段,并且都设为可选":

type LayerPatch = Partial<Pick<LayerConfig, "visible" | "sourceUrl">>;
// 等价于 { visible?: boolean; sourceUrl?: string; }
