第 4 课

Class 语法基础与 TS 的类型能力

目标:补齐 JS class 本身的语法,再叠加 TS 给 class 加的访问控制、参数属性和抽象类——为下一课的泛型打好地基。

为什么先讲 class,不直接进泛型

之前几课的例子都用 interface 描述"数据长什么样"(比如 VectorLayerConfig)。但如果一个东西不只是数据,还带有行为(方法)、需要被实例化、或者要表达"继承"关系,JS/TS 里通常用 class。泛型和工具类型的例子里经常会出现 class,所以这节课先把 class 本身(包括你之前自评"一般水平"的 JS class 语法)和 TS 给它加的类型能力一起讲清楚。

JS class 基础复习

这部分是 JavaScript 本身的语法,不是 TS 独有的:

class MapLayer {
  constructor(name, visible) {
    this.name = name;       // 实例字段
    this.visible = visible;
  }

  toggle() {                // 实例方法
    this.visible = !this.visible;
    console.log(`${this.name} 现在${this.visible ? "显示" : "隐藏"}`);
  }
}

const river = new MapLayer("river", true);
river.toggle(); // river 现在隐藏

继承用 extends + super: 

class VectorLayer extends MapLayer {
  constructor(name, visible, sourceUrl) {
    super(name, visible);   // 调用父类 constructor,先初始化 name/visible
    this.sourceUrl = sourceUrl;
  }
}

const roads = new VectorLayer("roads", true, "/data/roads.geojson");
roads.toggle(); // 继承自 MapLayer 的方法照样能用

TS 加的类型能力一:访问修饰符

JS 的字段默认谁都能从外部访问和修改。TS 加了三个访问修饰符,在编译期限制谁能访问某个字段/方法: 

class MapLayer {
  public name: string;       // 默认就是 public,随便写不写都一样
  private visible: boolean;  // 只能在 MapLayer 内部访问
  protected sourceUrl: string; // 能在 MapLayer 和它的子类里访问

  constructor(name: string, visible: boolean, sourceUrl: string) {
    this.name = name;
    this.visible = visible;
    this.sourceUrl = sourceUrl;
  }
}

const layer = new MapLayer("river", true, "/data/river.geojson");
layer.name;    // 合法
layer.visible; // 报错:Property 'visible' is private and only accessible within class 'MapLayer'.
和 [[0004]] 的 readonly 是同一套心智模型
private/protected 和 readonly 一样,只是编译期的访问控制,编译成 JS 之后这几个关键字直接被擦除,运行时字段和普通属性没有任何区别——也就是说如果用 (layer as any).visible 绕过 TS 检查,运行时照样能读写成功。如果你需要"运行时真正隐藏、外部绝对访问不到"的字段,要用 JS 原生的 #field 私有字段语法(这节课不展开,知道这个区别存在就够了)。

TS 加的类型能力二:参数属性(简写)

"在 constructor 参数前面加访问修饰符,自动声明字段并赋值"是很常用的简写: 

class MapLayer {
  constructor(
    public name: string,
    private visible: boolean,
    protected sourceUrl: string
  ) {}
  // 等价于上一节里手写的三个字段声明 + constructor 里的三行赋值
}

TS 加的类型能力三:抽象类(abstract class)

如果想表达"MapLayer 只是个模板,不该被直接实例化,具体怎么渲染由子类决定",用 abstract: 

abstract class MapLayer {
  constructor(public name: string) {}

  abstract render(): void; // 只声明方法签名,不写实现,子类必须自己实现

  describe(): void {
    console.log(`图层:${this.name}`); // 普通方法,子类直接继承
  }
}

class VectorLayer extends MapLayer {
  render(): void {
    console.log(`渲染矢量图层 ${this.name}`);
  }
}

new MapLayer("river");  // 报错:Cannot create an instance of an abstract class.
new VectorLayer("roads").render(); // 合法:渲染矢量图层 roads

这和第三课的可辨识联合是同一个问题("一堆不同形状的东西,怎么统一处理")的另一种解法:可辨识联合适合"纯数据 + if/switch 判断",抽象类适合"每种情况都有自己的一套方法实现,想让 TS 强制每个子类都必须实现某个方法"。

TS 加的类型能力四:implements

用 implements 让一个 class 保证满足某个 interface 的形状:

interface Renderable {
  render(): void;
}

class RasterLayer implements Renderable {
  constructor(public name: string) {}
  render(): void {
    console.log(`渲染栅格图层 ${this.name}`);
  }
  // 如果漏写 render 方法,TS 会报错:Class 'RasterLayer' incorrectly implements interface 'Renderable'.
}
