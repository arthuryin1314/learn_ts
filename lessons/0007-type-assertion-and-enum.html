第 7 课

类型断言(as)与枚举(enum)

目标:知道 as 什么时候该用、什么时候是在骗 TS;知道 enum 和字符串联合类型的区别,以及 enum 为什么和之前学的类型不一样——它在运行时是真实存在的。

类型断言(as):告诉 TS "相信我,这是这个类型"

第三课讲过类型收窄——TS 通过 if/typeof 等运行时检查自己判断出更精确的类型。但有些场景 TS 没办法自己推断出来,你比 TS 更清楚这个值实际是什么类型,这时候用 as 手动告诉 TS: 

function loadConfig(): unknown {
  return JSON.parse('{"name":"roads","visible":true,"sourceUrl":"/data/roads.geojson"}');
}

const raw = loadConfig();
raw.name; // 报错:Object is of type 'unknown'.

const config = raw as { name: string; visible: boolean; sourceUrl: string };
config.name; // 合法,TS 现在认为 config 是这个形状

as 只在编译期起作用,不会做任何运行时的检查或转换——它不像其他语言里的"类型转换"会真的改变值。如果断言错了,TS 编译期不会报错,但运行时可能直接崩: 

const raw2 = loadConfig();
const wrong = raw2 as { elevation: number };
wrong.elevation.toFixed(2); // 编译期不报错,运行时崩:Cannot read properties of undefined
和第三课的收窄是相反的思路
类型收窄是"用运行时检查换取 TS 的信任",as 是"不做检查,直接要求 TS 信任你"。能用收窄的地方优先用收窄,as 留给 TS 确实推断不出来、但你有充分把握(比如 JSON.parse 的返回值、DOM 查询结果)的场景。

as const:让 TS 推断出最精确的字面量类型

const status1 = "active";       // 类型推断为 string
const status2 = "active" as const; // 类型推断为字面量类型 "active"

let layerType = "vector";        // 类型是 string,后面可以随便改成别的字符串
const layerTypeConst = "vector" as const; // 类型是 "vector",不能再赋别的值

as const 常用在需要精确字面量类型的场景,比如给可辨识联合的 type 字段做标注,或者防止数组/对象被意外改动(as const 会让数组变成 readonly 元组)。

枚举(enum):一组命名常量

如果一个字段只能取几个固定值中的一个,之前的做法是字符串联合类型:

type LayerKind = "vector" | "raster" | "tile";
const kind: LayerKind = "vector";

TS 还提供了专门的语法 enum:

enum LayerKind {
  Vector,
  Raster,
  Tile,
}

const kind: LayerKind = LayerKind.Vector;
console.log(kind); // 0 —— 数字枚举默认从 0 开始编号

也可以用字符串枚举,值更直观、调试时更好读:

enum LayerKind {
  Vector = "vector",
  Raster = "raster",
  Tile = "tile",
}

const kind: LayerKind = LayerKind.Vector;
console.log(kind); // "vector"

关键区别:enum 在运行时是真实存在的对象

之前学的 interface、类型别名、泛型标注,编译成 JS 之后全部被擦除(参考第一课/第四课)。但 enum 不一样,它会被编译成一个真实的 JS 对象: 

enum LayerKind {
  Vector = "vector",
  Raster = "raster",
}

// 编译后大致相当于:
// const LayerKind = { Vector: "vector", Raster: "raster" };

Object.values(LayerKind); // ["vector", "raster"] —— 运行时真的能这样用

这也是为什么很多团队更倾向用字符串联合类型而不是 enum:联合类型是纯编译期概念,零运行时开销;enum 会往编译产物里加真实代码。两种方式都能达到"限制取值范围"的效果,enum 额外多了"运行时可以遍历/引用这个对象"的能力,但也多了一点运行时体积。

enum LayerKind {
  Vector = "vector",
