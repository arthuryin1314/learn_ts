第 3 课

联合类型与类型收窄

目标:能声明"多选一"的联合类型,并用类型收窄安全地处理每一种情况。

回顾:上节课埋的伏笔

上节课"动手练一下"的最后,让你给 Layer 加一个字段: 

readonly type: "vector" | "raster";

这里的 "vector" | "raster" 就是这节课的主角——联合类型(union type)。今天来搞清楚它到底是什么,以及为什么光声明还不够,还得配合"类型收窄"才能真正好用。

联合类型:表示"这个值只能是几种类型之一"

function formatFeatureId(id: string | number): string {
  // id 的类型是 "string 或者 number 之一"
  return String(id);
}

formatFeatureId("layer-1"); // 合法
formatFeatureId(42);        // 也合法
formatFeatureId(true);      // 报错:boolean 不在 string | number 里

| 读作"或"。string | number 表示这个值的类型必须是这两者之一,不能是别的类型,也不需要同时满足两者。

光有联合类型还不够

结合你熟悉的图层场景,假设矢量图层和栅格图层各有自己独有的配置字段: 

interface VectorLayerConfig {
  type: "vector";
  sourceUrl: string;
}

interface RasterLayerConfig {
  type: "raster";
  tileSize: number;
}

type LayerConfig = VectorLayerConfig | RasterLayerConfig;

function loadLayer(config: LayerConfig): void {
  console.log(config.sourceUrl);
  // 报错!Property 'sourceUrl' does not exist on type 'RasterLayerConfig'.
}

为什么会报错?因为 config 的类型是"两者之一",TS 不知道调用的这一刻它具体是哪一个——如果恰好是 RasterLayerConfig,那它根本没有 sourceUrl 这个字段。TS 在编译期就把这种"可能访问不存在的属性"的风险拦下来了。

类型收窄:typeof(适合基础类型)

把 formatFeatureId 写完整,需要先判断再使用:

function formatFeatureId(id: string | number): string {
  if (typeof id === "number") {
    return `#${id}`;          // 这个分支里,TS 知道 id 一定是 number
  }
  return id.toUpperCase();    // 这个分支里,TS 知道 id 一定是 string
}

这个过程叫类型收窄(narrowing):通过 if 判断,TS 会在每个分支里把联合类型"收窄"成更具体的那一种,让你能放心调用该类型独有的方法(比如 string 才有的 .toUpperCase())。

类型收窄:可辨识联合(适合对象)

回到图层配置的例子,同样的思路,判断条件换成公共字段 type:

function loadLayer(config: LayerConfig): void {
  if (config.type === "vector") {
    console.log("加载矢量图层:", config.sourceUrl); // 安全,这里收窄成 VectorLayerConfig
  } else {
    console.log("加载栅格图层, tileSize:", config.tileSize); // 安全,这里收窄成 RasterLayerConfig
  }
}
可辨识联合(discriminated union)
像 type: "vector" 这种每个分支取值都不同、且是字符串字面量类型的公共字段,叫"辨识字段"(discriminant)。有了它,TS 只要看一眼这个字段的值,就能在对应分支里精确推断出完整的类型——这是处理"一个变量可能是好几种不同形状的对象"时最常用、最推荐的模式。

动手练一下

在 TypeScript Playground 里试试:

interface PointFeature {
  kind: "point";
  coordinates: [number, number];
}

interface LineFeature {
  kind: "line";
  coordinates: [number, number][];
}

type MapFeature = PointFeature | LineFeature;

function describeFeature(feature: MapFeature): string {
  if (feature.kind === "point") {
    return `点坐标: ${feature.coordinates[0]}, ${feature.coordinates[1]}`;
  } else {
    return `线段包含 ${feature.coordinates.length} 个点`;
  }
}

console.log(describeFeature({ kind: "point", coordinates: [116.4, 39.9] }));

跑通之后自己试着:1) 把 if 分支里的收窄判断删掉,直接写 feature.coordinates[0],看 TS 会不会报错、报的什么错;2) 加一个 PolygonFeature(kind: "polygon",coordinates: [number, number][][]),扩展联合类型和 describeFeature 的判断分支。
